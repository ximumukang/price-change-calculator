# 涨跌幅计算应用 - 新功能灵感清单

> 基于当前项目已实现功能（用户认证、涨跌幅 CRUD、RSA+JWT 安全体系），整理的可扩展方向。

---

## 🔥 高价值 & 实用性强

### 1. 数据可视化看板 📊

当前 Dashboard 只有表格，加一个涨跌幅图表（柱状图/环形图/折线图），用 ECharts 展示各标的涨跌幅分布、盈亏比例等。前后端改动小，视觉效果拉满。

- **技术方案**: 前端引入 ECharts，新增 Dashboard 图表区域
- **改动范围**: 前端为主，后端可复用现有列表 API

### 2. 价格追踪 & 历史趋势 📈

为 PriceItem 增加历史价格记录表 `price_history`，每次更新时自动保存旧值。前端用折线图展示某个标的的价格变化趋势——这才是"涨跌幅计算"的核心价值。

- **技术方案**: 新增 `price_history` 表 + 后端 API + 前端折线图
- **改动范围**: 前后端均有较大改动，是项目升级的关键一步

### 3. 分页 & 搜索筛选 🔍

当前列表全量返回 + 只能按涨跌幅排序。加个分页、名称搜索、日期范围筛选、涨跌幅区间筛选。这是**数据量增长后的刚需**。

- **技术方案**: 后端 MyBatis-Plus 分页 + 条件构造器，前端 Element Plus 分页组件
- **改动范围**: 后端分页 API 改造，前端列表页升级

---

## 💡 产品体验提升

### 4. 分类/标签管理 🏷️

给 PriceItem 加分类（如：股票/基金/加密货币/房产），支持按分类筛选和统计汇总。新增 `category` 字段 + 前端 Tab 切换。

- **技术方案**: PriceItem 表加 `category` 字段，后端筛选 API，前端 Tab 组件
- **改动范围**: 中等，前后端均需改动

### 5. 个人设置 & 修改密码 ⚙️

当前无修改密码、注销账号等基础用户管理功能。后端加 `PUT /api/auth/password`，前端加 Settings 页面。

- **技术方案**: 新增密码修改 API + Settings 页面
- **改动范围**: 较小，但属于基础功能补全

### 6. 暗黑模式 🌙

Element Plus 原生支持，改动量小但体验提升明显。

- **技术方案**: 使用 Element Plus 的 dark mode + CSS 变量切换
- **改动范围**: 纯前端，改动量小

---

## 🛡️ 技术补完

### 7. 后端单元/集成测试 🧪

当前 `src/test/` 完全为空，是最大的技术债务。重点覆盖：涨跌幅计算精度、除零保护、Token 过期验证、账户锁定逻辑。

- **技术方案**: 使用 `@SpringBootTest` + `@MockBean` 编写测试
- **改动范围**: 纯后端，重点在 Service 和 Security 层

### 8. Token 注销 & 黑名单 🔒

用户退出后 Token 仍在有效期内可用（安全隐患）。可用 Redis 或内存 Set 实现 Token 黑名单，jti 已预留好。

- **技术方案**: 登出时将 Token jti 存入黑名单，JWT 过滤器检查黑名单
- **改动范围**: 后端为主，需引入缓存机制

---

## 🚀 进阶玩法

### 9. 多用户分享 & 对比 👥

允许用户将自己的涨跌幅列表分享给其他用户查看，或做投资组合对比。需要加 `share_token` 机制和分享 API。

- **技术方案**: 新增分享链接机制 + 分享 API + 公开访问页面
- **改动范围**: 较大，涉及新的数据模型和权限设计

### 10. 智能提醒 🔔

当某个标的涨跌幅达到设定阈值时，通过浏览器 Notification 或邮件通知。后端用 Spring Scheduler 定时检查，前端用 Web Push API。

- **技术方案**: 后端定时任务 + 阈值配置 + 前端通知
- **改动范围**: 较大，前后端均需新增模块
